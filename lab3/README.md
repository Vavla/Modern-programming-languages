Реализована часть CONSUMER, работа совместно с Портновой Лидией (у неё суперский producer)

Описание реализации:
1. Producer отправляет сообщение в виде словаря {'table_name': name, 'columns': columns_list, 'rows': rows_list}
2. При получении сообщения из него выделяются все данные и проверяются на то, что они не пустые
3. Потом проверяется наличие в базе данных таблиц с указанным именем. Если такой нет, она сначала создаётся, потом заполняется. Если есть, этап создания упускается.
4. При создании таблицы происходит проверка типа каждого столбца помимо id. В нашей реализации проверка идёт только на 3 типа: целое число, число с плавающей точкой и еще строка.
5. При создании и заполнении таблицы формируются строки запроса, который потом отправляется на выполнение в базу данных.


Используются модули kafka, psycopg2(для postgresql)
